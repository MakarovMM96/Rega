# Инструкции по сборке и запуску приложения

## Обновленное приложение без AI-функций

В этом репозитории были удалены все AI-функции, чтобы обеспечить стабильную работу приложения без необходимости в API-ключах. Вот что было изменено:

1. Удален файл `/services/geminiService.ts`
2. Из компонента `App.tsx` удален импорт функции `generateHypeMessage`
3. Из `package.json` удалена зависимость `@google/genai`
4. Обновлена логика в `handleSubmit`, чтобы не использовать AI-функции
5. Интерфейс обновлен, чтобы отражать использование подготовленного сообщения вместо AI-сообщения
6. Обновлен `README.md`, чтобы отразить изменения

## Подготовка к установке

Перед установкой убедитесь, что у вас установлены:
- Node.js (версия 16 или выше)
- npm или yarn

## Установка зависимостей

1. Удалите старые файлы зависимостей, если они есть:
```bash
rm -rf node_modules package-lock.json
```

2. Установите зависимости:
```bash
npm install --no-audit --no-fund --prefer-offline
```

Если у вас возникли проблемы с памятью, вы можете попробовать:
```bash
npm install --no-audit --no-fund --prefer-offline --no-optional
```

## Запуск приложения

Для запуска приложения в режиме разработки выполните:
```bash
npm run dev
```

Приложение будет доступно по адресу http://localhost:5173 (или другому порту, указанному в терминале)

## Сборка для продакшена

Для создания продакшен-сборки выполните:
```bash
npm run build
```

## Функциональность приложения

После удаления AI-функций приложение по-прежнему полностью функционально:

- Форма регистрации работает корректно
- Все валидации полей сохранены
- Данные отправляются через сервис yandexService (в демонстрационном режиме)
- Вместо AI-сообщения показывается подготовленное приветственное сообщение с использованием данных пользователя
- Интерфейс адаптивен и сохраняет весь визуальный дизайн

## Возможные проблемы и решения

1. Если возникает ошибка "Cannot allocate memory" при установке:
   - Попробуйте установить зависимости по отдельности
   - Используйте флаги `--no-audit --no-fund --prefer-offline`
   - Увеличьте размер swap-файла, если возможно

2. Если приложение не запускается:
   - Проверьте версию Node.js (должна быть 16 или выше)
   - Убедитесь, что порт 5173 не занят другим приложением
   - Проверьте, что все зависимости установлены корректно

3. CORS-ошибки при работе с Yandex API:
   - Это нормальное поведение в демонстрационной версии
   - В продакшен-окружении для полноценной работы с Yandex API потребуется backend-сервер
